**What it is:** A regularly-updated archive of extracted system prompts across vendors — including Claude Code (Opus 4.8/4.6), Claude Fable 5, Opus 4.8, plus the Claude Code Glob & Grep tool definitions. Featured in the Washington Post.
**Reusable for us:** A **reference** for how the Claude Code harness actually instructs the model (system prompt + tool defs). Useful when reasoning about why the agent behaves a certain way, what the built-in tools promise, and how to phrase commands/hooks to align with — not fight — the harness prompt.
**Token / effectiveness angle:** Indirect: knowing the real system prompt lets us avoid redundant instructions (don't restate what the harness already enforces) — a token saving in our own commands/CLAUDE.md.
**How to adopt:** Bookmark as a reference source (no asset to vendor; contents shift and are third-party/leaked, treat as informational only). Catalog under reference/MCP-adjacent. Recheck the Claude Code prompt entries when behavior surprises us.
